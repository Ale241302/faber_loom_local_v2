3. Sin emoji en producto: El coral es el unico "emoji" que necesitamos.
4. Sin exclamaciones: La confianza no necesita signos de admiracion.
5. Respeto al oficio: Nunca asumir que el usuario no sabe. Nunca explicar lo obvio.

## 4. Conclusiones para FaberLoom

### Principio 1: La restriccion es distincion
Las referencias de mayor calidad (Linear, Resend, Plain) usan 1-2 colores de acento maximo. FaberLoom tiene exactamente dos: coral para accion y pizarra para estado/info. No necesitamos mas.

### Principio 2: La tipografia es identidad
La combinacion Inter + Crimson Pro Italic 500 no tiene equivalente directo en las referencias. Es el diferenciador silencioso: funcional como Linear, distinguida como Resend, memorable como ninguna otra.

### Principio 3: La densidad es respeto
Linear demuestra que se puede mostrar mucha informacion sin abrumar. La clave: jerarquia clara, espacio proporcional, acciones contextuales. El usuario senior valora la densidad informativa; el novato valora la claridad. Ambos son compatibles con buen diseno.

### Principio 4: El producto habla por si mismo
Framer y Resend enseñan que la mejor landing page es el producto visible. FaberLoom no necesita explicar por que es mejor; necesita mostrar un hilo de comunicacion tan claro que la mejora sea evidente.

### Principio 5: El calmo no es lento
Raycast y Cursor demuestran que "calmo" describe la sensacion, no la velocidad. Las microinteracciones rapidas (~150ms), los shortcuts visibles, la command palette: todo esto hace que FaberLoom se sienta rapido y tranquilo al mismo tiempo.
