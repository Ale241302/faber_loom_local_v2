## Reglas de creación
- Estructura Claim+Subhead: CLAIM = emocional grande + SUBHEAD = técnico menor
- Secuencia 5 bullets: Hook → Problema → Evidencia → Lifestyle → Garantía
- Hooks por producto (cerrados, no inventar)
- SEO A10+Rufus: títulos 70-80 chars, backend 250 bytes
- Compliance SIGIS en copy
- A+ Content: 5 módulos (Hero, Tech, Comparativa, Lifestyle, Brand Story)
- Producción visual IA
- GEO (Generative Engine Optimization)

## Adaptación i18n (3 capas)
1. Nunca traducir: tech names, marca, sello (ref → POL_NUNCA_TRADUCIR)
2. Adaptar: claims, subheads, specs descriptivos (adaptación creativa, no literal)
3. Convertir: unidades de medida según mercado (ref → ENT_MERCADO_{M})
