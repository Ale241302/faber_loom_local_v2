# Kimi Swarm #5 — Plan de Ejecución

## Objetivo
Producir output dual obligatorio para Muito Work Limitada:
1. `ENT_GOB_TEAM_INVENTORY_v2` — roles SOTA por dominio
2. `ENT_GOB_ENGINEERING_COMPETENCIES_v1` — competencias transversales

## Formato de Output
- Español 100%, técnico directo, sin relleno
- Tablas > prosa, bullets > párrafos
- Citaciones inline con confianza explícita [HIGH]/[MEDIA]/[BAJA]
- Máximo 350 palabras por sub-agente
- Anti-patterns 2026 obligatorios
- Sin disclaimers

## Stage 1 — Investigación Paralela (12 Sub-agentes)
Cada sub-agente especializado recibe:
1. Contexto completo de MWT (3 productos, constraints, inventario v1.0, gaps)
2. Un dominio A1-A12 con sus 6 preguntas específicas
3. Formato exacto de salida (6 bloques por dominio)
4. Instrucción de búsqueda web para verificar SOTA 2025-2026
5. Límite de 350 palabras

Agentes:
- A1_ARCH — Arquitectura de software moderna
- A2_PLATFORM_ENG — Platform Engineering / IDP / DevEx
- A3_AI_AGENTS — Multi-agent / agentic frameworks SOTA
- A4_FRONTEND — Frontend SOTA 2026
- A5_BACKEND — Backend Python / FastAPI / async
- A6_DATA — Data / lakehouse / streaming / vector DB
- A7_OBSERV — Observability / SLO / distributed tracing
- A8_SECURITY — Security / supply chain / DevSecOps
- A9_COMPLIANCE — Compliance LATAM 2026 / privacy engineering
- A10_HW_IOT — Hardware embedded / BLE / OTA / device management
- A11_MOBILE — Mobile iOS/Android 2026
- A12_QA_REL — QA / testing / chaos / contract / reliability eng

## Stage 2 — Síntesis Integrada
Después de recibir los 12 outputs:
- S1: TOP-10 roles indispensables que NO están en v1.0 o están sub-dimensionados
- S2: TOP-10 competencias transversales
- S3: TOP-5 anti-patterns 2026 a evitar
- S4: TOP-5 inversiones de plataforma con más ROI en 24m
- S5: Meta-gaps del inventario v1.0 refutados/sub-dimensionados/sobre-dimensionados

## Stage 3 — Ensamblaje Final
Integrar todo en un único output ≤30 KB en formato markdown.
