## Caveats técnicos

- **Instalación múltiple de `coreyhaines31/marketingskills`**: El repo contiene 30+ skills en subdirectorios. Se puede instalar el repo completo con `npx skills add coreyhaines31/marketingskills` (instala todas) o skill individual con `--skill <nombre>`. Las skills se escriben en `~/.claude/skills/<skill-name>/` o en el directorio de skills del proyecto local (`.claude/skills/`).
- **`AgriciDaniel/claude-seo`**: Requiere Python 3.10+. El script `install.sh` detecta Claude Code y escribe en `~/.claude/skills/seo/` y `~/.claude/skills/seo-*/`. Si ya existe `anthropic-skills/seo`, podría haber conflicto de paths — verificar antes. Requiere Node.js para algunas herramientas de análisis. Extensiones DataForSEO, Firecrawl y Banana requieren API keys adicionales.
- **`aaron-he-zhu/seo-geo-claude-skills`**: Es un monorepo con 20 skills organizadas por directorios (`research/`, `build/`, `optimize/`, `monitor/`, `cross-cutting/`). Instalación vía `npx skills add aaron-he-zhu/seo-geo-claude-skills` instala todas. Usa shell scripts 100% (no Python). Requiere compatible con Agent Skills spec.
- **`nexscope-ai/Amazon-Skills`**: 51 skills, muchas en Beta (`🔶`). Las de Listing Optimization están en Beta excepto `amazon-listing-optimization` (✅ Available). No requiere API keys para el core (prompt-driven). Las script-driven (fba-calculator, keyword-research) usan Python stdlib.
- **`CosmoBlk/email-marketing-bible`**: Instalación manual via `git clone` a `~/.claude/skills/email-marketing-bible`. Es un solo archivo SKILL.md masivo (~68K palabras). Puede ralentizar el context window si se carga completo; verificar que Claude Code lo carga por fragmentos.
- **`BrianRWagner/ai-marketing-openclaw-skills`**: Instalación manual vía `cp -R skills/* ~/.codex/skills/` (o `~/.claude/skills/`). Diseñado para OpenClaw/Codex pero compatible con Claude Code. No usa `npx skills` — instalación manual únicamente.
- **`smixs/creative-director-skill`**: No requiere instalación CLI — copiar carpeta `creative-director/` al proyecto o skills directory. Usa wikilinks `[[...]]` para referenciar archivos internos; compatibilidad depende del agente.
- **`alirezarezvani/claude-skills`**: Usa sistema de plugins Claude Code (`/plugin marketplace add` + `/plugin install`). Compatible con 12 herramientas. Las skills de marketing son 43, organizadas en 7 "pods". Requiere leer `marketing-ops/SKILL.md` primero (router). Los 32 Python tools son stdlib-only (zero pip installs).
- **Conflictos potenciales**: `seo-audit` existe tanto en `coreyhaines31/marketingskills` como en `AgriciDaniel/claude-seo` (como `/seo audit`). Si se instalan ambos, el agente puede confundirse sobre cuál activar. Se recomienda instalar **solo uno** para SEO: `AgriciDaniel/claude-seo` para depth técnico, o `coreyhaines31/seo-audit` para lightweight + complementar con `aaron-he-zhu/seo-geo-claude-skills`.
- **Product-marketing-context**: Todas las skills de `coreyhaines31/marketingskills` leen `.agents/product-marketing-context.md` (o `.claude/product-marketing-context.md`) antes de actuar. **Es crítico crear este documento primero** para que las skills funcionen correctamente.
- **No hay skills nativas en español/portugués**: Todas las skills encontradas están en inglés. El output se generará en inglés a menos que el usuario especifique español en el prompt. Para MWT/FB en LATAM, esto es una limitación significativa que requiere prompts de contexto explícitos.
