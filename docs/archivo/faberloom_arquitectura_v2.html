✍️ Voice Profile
Construido de enviados históricos. 12 atributos: formalidad, longitud, cierre, muletillas, tono por tipo de contacto. Se afina con cada corrección del usuario.
Haiku · por usuario
TABLERO COVEY — prioridad por urgencia × peso estratégico del sender
Q1 · Atender ahora
Urgente + sender estratégico. Draft listo con contexto completo. Acción inmediata.
Q2 · Planificar
No urgente + estratégico. Respuesta profunda. El sistema sugiere cuándo agendar tiempo.
Q3 · Respuesta corta
Urgente + bajo peso. Draft mínimo o sugerencia de delegar. 1 clic para aprobar.
Q4 · Archivar
Sin urgencia ni peso estratégico. Clasificado automáticamente. Sin acción requerida.
🔄 Feedback Loop
Cada movimiento del usuario entrena el sistema: movió Q3→Q1, editó el draft, archivó sin leer. Sin formulario. Sin configuración. Aprende solo con el uso.
Reinforcement · tiempo real

⑤ Knowledge Base — el cerebro del negocio, granular y privado
Capa 1 · PERSONAL
Solo el usuario. Decisiones, notas, historial propio. Nunca sale al equipo ni a los agentes de otros roles.
Capa 2 · ROL
Comparte con mismo rol. Procesos de área, playbooks de equipo.
Capa 3 · ORGANIZACIÓN
Todo el tenant. Promovido con aprobación del admin. Fichas técnicas, políticas, procedimientos.
Capa 4 · PÚBLICO/VIP
Catálogos, FAQs, fichas de producto. Accesible por externos en el futuro.
⬆ Flujo de conocimiento
Usuario crea en Capa 1 → propone "subir a organización" → Haiku clasifica y sugiere visibilidad → admin aprueba → sube a Capa 3/4. Default siempre restrictivo. El conocimiento se cocina en lo personal y se promueve cuando está maduro.
